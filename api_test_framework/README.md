## 🚀 Quick Start

### 1. Clone & install

```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
cd api_test_framework
pip install -r requirements.txt
```

### 2. Configure your environment

```bash
cp .env.example .env
# Edit .env with your API credentials
```

Or set environment variables directly:

```bash
export ENV=dev
export API_TOKEN=your_token_here
export API_KEY=your_api_key_here
```

### 3. Run tests

```bash
# All tests
make test

# By marker
make smoke
make regression
make performance

# Specific environment
ENV=staging pytest -m smoke -v

# Specific test file
pytest tests/test_suites/test_smoke.py -v
```

## ✅ Writing Tests

Tests use a clean, chainable validator API:

```python
from core.validator import validate
from utils.data_helpers import make_post_payload

def test_create_post(api_client):
    payload = make_post_payload()
    response = api_client.post("/posts", json=payload)

    (validate(response)
        .status_created()           # assert 201
        .content_type_json()        # assert Content-Type
        .body_contains_key("id")    # assert key exists
        .body_value_equals("title", payload["title"])   # assert value
        .response_time_under(2000)) # assert < 2s
```

### Available validators

| Method | Description |
|---|---|
| `.status_code(n)` | Assert exact HTTP status |
| `.status_ok()` | Assert 200 |
| `.status_created()` | Assert 201 |
| `.status_in([200, 201])` | Assert status is one of |
| `.body_contains_key("key")` | Assert key exists in JSON |
| `.body_value_equals("key", val)` | Assert key == value |
| `.body_value_not_null("key")` | Assert key is not null |
| `.body_is_list()` | Assert response is a JSON array |
| `.body_list_not_empty()` | Assert array has at least one item |
| `.body_matches_schema(schema)` | Assert JSON Schema compliance |
| `.header_equals(h, val)` | Assert exact header value |
| `.header_contains(h, sub)` | Assert header contains substring |
| `.content_type_json()` | Assert Content-Type: application/json |
| `.response_time_under(ms)` | Assert response time in milliseconds |
| `.dump()` | Log full response for debugging |

## 🔐 Authentication

Set in `config/<env>.yaml` or via environment variables:

```yaml
# Bearer token
auth_type: bearer
token: ""   # or set API_TOKEN env var

# API Key
auth_type: api_key
api_key: ""  # or set API_KEY env var
```

For **OAuth2 client credentials**:

```python
from core.auth import AuthManager

token = AuthManager.fetch_oauth2_token(
    token_url="https://auth.example.com/oauth/token",
    client_id="your_client_id",
    client_secret="your_client_secret",
)
```

## 📊 Reports

After each run, reports are saved in `reports/`:

| Report | File | How |
|---|---|---|
| **HTML** | `reports/report.html` | Auto-generated by pytest-html |
| **JSON** | `reports/report.json` | Auto-generated by pytest-json-report |
| **Allure** | `allure-report/` | Run `allure serve allure-results` |
| **Logs** | `logs/test_run.log` | Rotating, 5 MB max |

## ⚙️ CI/CD — GitHub Actions

The workflow runs on every push and PR, plus daily at 06:00 UTC.

**Setup secrets in GitHub → Settings → Secrets and variables → Actions:**

| Secret | Description |
|---|---|
| `API_TOKEN` | Bearer token for staging/prod |
| `API_KEY` | API key for staging/prod |

Trigger a manual run from the **Actions** tab and choose your environment + marker.
